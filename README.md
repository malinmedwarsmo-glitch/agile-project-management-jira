#  Agile Project Management with Jira

> A simulated IT project demonstrating real-world Agile/Scrum workflows using Jira, Confluence, Miro, and GitHub.

---

## Project Overview

This project simulates an **IT Transformation initiative** managed with Agile methodologies.  
The goal was to practice and demonstrate hands-on experience with industry-standard tools used in IT departments, consulting firms, and tech companies.

## Tools & Skills Demonstrated

###  Jira – Project & Task Management
- Created **Epics, User Stories, Tasks, and Bugs**
- Planned and executed **2 Sprints**
- Used **Kanban Board** and **Backlog Refinement**
- Tracked velocity and sprint progress

###  Confluence – Project Documentation
- **Project Plan** – Scope, goals, timeline
- **Meeting Notes** – Sprint Planning & Retrospective
- **Risk Register** – Identified and assessed project risks
- **Stakeholder Analysis** – Stakeholder mapping and communication plan

###  Miro – Visual Planning
- **Process Diagram** – As-Is vs. To-Be workflow
- **Sprint Planning Board** – Visual backlog and sprint board

###  GitHub – Repository & Documentation
- Structured project documentation
- Screenshots and deliverables
- Version-controlled project files

##  Jira Board Structure

```
Epic: IT System Modernization
│
├── User Story: As a project manager, I want a centralized dashboard...
│   ├── Task: Set up Jira project and board
│   ├── Task: Define sprint goals
│   └── Bug: Fix incorrect status labels on board
│
├── User Story: As a stakeholder, I want regular status updates...
│   ├── Task: Create Confluence project plan page
│   └── Task: Write meeting notes template
│
└── User Story: As a team member, I want clear task assignments...
    ├── Task: Define team roles and responsibilities
    └── Task: Create stakeholder analysis document
```
